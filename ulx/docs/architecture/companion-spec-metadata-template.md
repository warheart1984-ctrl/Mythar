# Companion Spec Metadata Template

## Purpose
This template defines the standard metadata block that every companion specification SHOULD include.

It keeps companion specifications machine-readable, versioned, and consistently stewarded without turning the template itself into a governance layer.

## Scope
This template covers:
- document status
- purpose
- scope
- normative dependencies
- relationship to CIS Core
- one authoritative responsibility
- version
- stewardship

This template does not cover:
- constitutional obligations
- runtime execution
- policy decision making
- registry promotion rules

## Standard Metadata Block
Every companion specification SHOULD include the following metadata block:

- Document Status
- Purpose
- Scope
- Normative Dependencies
- Relationship to CIS Core
- One Authoritative Responsibility
- Version
- Stewardship

## Usage Rules
The metadata block SHOULD:
- appear near the top of the document
- use the same field names across companion specifications
- stay descriptive rather than normative about CIS Core itself
- identify one authoritative responsibility for the document

The metadata block MUST NOT:
- replace the specification's own normative scope
- redefine CIS Core authority
- add runtime behavior
- collapse companion specifications into the constitutional core

## Relationship to CIS Core
Companion specifications remain subordinate to CIS Core for constitutional authority.

The metadata block makes that relationship explicit while preserving implementation-neutral documentation.

## Required Artifact Pairing
This template SHOULD be accompanied by:
- a companion specification that uses the block
- a docs-site mirror page
- a conformance artifact or governed stewardship review record

## Stewardship Notes
This template is intended to make every companion specification easier to compare, review, and govern across the ecosystem.
