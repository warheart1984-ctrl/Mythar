# Stewardship Review Checklist

## Purpose
This checklist defines the review questions that every major engineering slice SHOULD answer before it is considered complete.

It keeps stewardship evidence-oriented and replayable without replacing the constitutional authority of CIS Core.

## Scope
This checklist covers:
- CIS requirement mapping
- companion specification ownership
- evidence production
- conformance validation
- replay and independent verification

This checklist does not cover:
- constitutional rule changes
- runtime policy adjudication
- artifact promotion outside the governed process

## Completion Questions
Every major engineering slice SHOULD answer all five questions:

1. Which CIS requirement does this implement?
2. Which companion specification owns it?
3. What evidence does it produce?
4. Which conformance tests validate it?
5. How is it replayed and independently verified?

## Review Standard
A slice SHOULD be considered complete only when:
- the implementing requirement is named explicitly
- the owning companion specification is identified
- the evidence package is attached or referenced
- the conformance path is named
- the replay or verification path is documented

## Stewardship Rules
Reviewers SHOULD:
- prefer evidence over assertion
- prefer traceability over summary prose
- preserve CIS Core as the constitutional authority
- keep companion-spec responsibilities distinct
- reject slices that cannot be replayed or independently verified

## Required Artifact Pairing
This checklist SHOULD be accompanied by:
- a companion-spec metadata template
- a standards traceability reference
- a launch-readiness reference
- a conformance review record

## Stewardship Notes
The checklist is intended to be used as a review gate for implementation slices, companion specifications, and operational handoffs.
